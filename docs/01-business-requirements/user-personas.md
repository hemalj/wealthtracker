# User Personas

## Overview

This document defines the primary user personas for WealthTracker, their needs, pain points, and how the product serves them.

## Primary Personas

### 1. The Organized Investor - "Sarah"

**Demographics**
- Age: 32
- Occupation: Software Engineer
- Income: $150K
- Investment Experience: 5+ years
- Tech Savviness: High

**Investment Profile**
- 3 brokerage accounts (Fidelity, Robinhood, Interactive Brokers)
- Portfolio Value: $250K
- Asset Mix: 70% stocks, 20% ETFs, 10% crypto
- Trading Frequency: 5-10 transactions per month
- Investment Style: Buy and hold with tactical trading

**Goals & Motivations**
- Track overall portfolio performance across all accounts
- Accurate cost basis for tax reporting
- Understand sector allocation and diversification
- Monitor dividend income streams
- Prepare for tax season efficiently

**Pain Points**
- Uses spreadsheets that are hard to maintain
- Missing historical transactions from old accounts
- Can't easily see total portfolio performance
- Time-consuming to update prices manually
- No way to analyze sector/geography exposure

**How WealthTracker Helps**
- Single dashboard showing all accounts
- Transaction-level tracking for accurate cost basis
- Automated holdings summary by sector/account
- Dividend tracking and projections
- Export capabilities for tax preparation

**Key Features Used**
- Portfolio dashboard (daily)
- Transaction management (weekly)
- Holdings reports (weekly)
- Dividend tracking (monthly)
- Bulk transaction import (quarterly)

## User Journey Maps

### Sarah's Journey (Organized Investor)

**Week 1: Discovery & Onboarding**
1. Finds WealthTracker through Google search for "portfolio tracker"
2. Signs up with Google account
3. Creates first account (Fidelity)
4. Manually enters 5 current holdings
5. Explores dashboard and features
6. Adds second account (Robinhood)

**Week 2-4: Active Setup**
1. Imports historical transactions from broker CSV
2. Resolves symbol mapping issues
3. Adds dividend transactions
4. Verifies cost basis matches broker
5. Explores holdings and performance reports
6. Invites spouse to share dashboard (future feature)

**Month 2+: Steady State**
1. Checks dashboard 2-3x per week
2. Adds transactions after each trade
3. Reviews monthly performance reports
4. Uses dividend calendar for planning
5. Exports data for tax prep (Q4)

---

### Robert's Journey (Hands-On Retiree)

**Week 1: Discovery & Onboarding**
1. Recommended by friend or financial advisor
2. Signs up with email
3. Creates Vanguard account
4. Manually enters 15 dividend stocks
5. Adds IRA account
6. Reviews simple holdings report

**Month 1-3: Historical Data Entry**
1. Gradually enters old transactions (as time permits)
2. Focuses on accurate cost basis
3. Adds dividend history
4. Reviews dividend income summaries
5. Prints reports for advisor meeting

**Ongoing Use**
1. Checks dashboard weekly
2. Adds transactions quarterly (after trades)
3. Records dividends monthly
4. Reviews annual performance
5. Shares reports with advisor

## Use Case Scenarios

### Scenario 1: Tax Season Preparation
**User**: Sarah (Organized Investor)
**Goal**: Export all transactions and gains/losses for tax filing

**Steps**:
1. Navigate to Transactions page
2. Filter by date range (Jan 1 - Dec 31, 2025)
3. Filter by transaction type (Sell only)
4. Export to CSV
5. Review realized gains/losses report
6. Export summary for tax software

**Success Criteria**:
- Accurate cost basis for all sales
- Realized gains calculated correctly
- Easy export format for tax software
- Complete transaction history

---

### Scenario 2: Portfolio Rebalancing
**User**: Mike (Active Trader)
**Goal**: Analyze sector allocation and rebalance

**Steps**:
1. View dashboard holdings summary
2. Review allocation by sector
3. Identify overweight sectors
4. Filter transactions by symbol
5. Plan rebalancing trades
6. Execute trades in broker
7. Record transactions in WealthTracker

**Success Criteria**:
- Clear sector allocation visualization
- Easy identification of imbalances
- Quick transaction entry after rebalancing
- Updated portfolio view immediately

---

### Scenario 3: Dividend Income Planning
**User**: Robert (Hands-On Retiree)
**Goal**: Project quarterly dividend income

**Steps**:
1. Navigate to Dividend dashboard
2. Review historical dividend income by quarter
3. See upcoming dividend schedule
4. Calculate projected annual income
5. Compare to budget needs
6. Export dividend report for records

**Success Criteria**:
- Accurate historical dividend data
- Clear quarterly income trends
- Dividend calendar with estimates
- Easy export for financial planning

## Persona-Driven Feature Prioritization

### Must Have (MVP) - Serves All Personas
- User authentication (email + Google)
- Account management (CRUD)
- Transaction management (CRUD, all types)
- Basic holdings dashboard
- Transaction table with search/filter
- Simple dividend tracking

### Should Have (Post-MVP) - Power Users
- Bulk transaction import/export
- Advanced performance analytics
- Realized/unrealized gains reports
- Multi-currency support
- Stock splits handling
- Historical cost basis tracking

### Could Have (Future) - Advanced Features
- Portfolio sharing with advisors
- Mobile app
- Automated broker synchronization
- Tax optimization recommendations
- Benchmark comparison
- Asset allocation suggestions

### Won't Have (Yet) - Out of Scope
- Live trading integration
- Automated tax filing
- Financial advisor marketplace
- Social/community features
- Cryptocurrency wallets
- Futures/options advanced analytics
